# 资产账户编辑与快照设计

## 现状决策

- 资产账户只有一个维护入口：新增/编辑账户。
- 金额是账户的普通字段，和名称、类型、归属、备注一起编辑。
- 不再提供单独的“调整”按钮、调整弹窗或调整记录列表。
- 修改账户金额不会自动保存历史记录。
- 资产历史通过“保存快照”手动记录，每次快照保存当前所有未删除账户的金额。

## API 行为

- `PATCH /api/accounts/:id`
  - 请求体包含 `name`、`type`、`ownerName`、`currentValue`、`note`
  - 行为：更新账户基础信息和当前金额
  - 不写入 `AccountSnapshot`
- `POST /api/accounts/snapshots`
  - 行为：为当前所有资产账户保存一条当日快照
  - 用途：作为资产趋势和历史记录的数据来源
- `GET /api/accounts/:id/snapshots`
  - 行为：查询该账户的手动快照记录

## 前端交互

- 资产账户操作列只保留“编辑”和“删除”。
- 编辑 Drawer 显示“当前金额”，编辑账户时可直接修改金额。
- 资产账户页保留“保存快照”按钮。
- 当前金额在表格中只做展示，不弹出“最近调整”记录。
